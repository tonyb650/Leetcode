### 202. Happy Number

**Problem**\
Write an algorithm to determine if a number `n` is happy.

A **happy number** is a number defined by the following process:

- Starting with any positive integer, replace the number by the sum of the squares of its digits.
- Repeat the process until the number equals 1 (where it will stay), or it **loops endlessly in a cycle** which does not include 1.
- Those numbers for which this process **ends in 1** are happy.

Return `true` if `n` is a happy number, and `false` if not.

**Example 1:**\
Input: n = 19
Output: true
Explanation:
12 + 92 = 82
82 + 22 = 68
62 + 82 = 100
12 + 02 + 02 = 1

**Example 2:**\
Input: n = 2\
Output: false

**Constraints:**
- `1 <= n <= 231 - 1`

### Solution, Recursion
Runtime 2ms Beats 38.37%\
Memory 57.39MB Beats 12.71%
```js
var isHappy = function(n, s = new Set()) { // Might use less memory as a while loop?
    const m = String(n).split("").reduce((a,c) => a + Number(c)**2, 0) // I wonder if it would be faster without casting to a string
    if(m === 1) return true
    if(s.has(m)) return false
    s.add(m)
    return isHappy(m, s)
};
```

var isHappy = function(n) {
    if (n === 1) return true
    const s = new Set()
    while (!s.has(n)) {
        s.add(n)
        let m = n
        n = 0
        while (m) {
            n += (m % 10)**2
            m = ~~(m/10)
        }
    }
    if (n === 1) return true
    return false
};
